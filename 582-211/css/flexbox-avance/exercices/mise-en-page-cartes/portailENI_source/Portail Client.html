Créer une mise en page avec des cartes

Le fichier d’exemple à télécharger se nomme : 01-Exemples Flexbox/C06-C.html.

1. Les affichages de la page

Nous allons créer une page proposant un affichage avec des cartes (cards layout en anglais). Ce type de présentation est très classique dans les sites d’e-commerce, dans les portfolios et tous les sites qui ont besoin d’afficher des informations succinctes en pavés ou en mosaïque. Les solutions précédentes à l’avènement du module Flexbox étaient lourdes à mettre en place et généraient beaucoup de problèmes. Avec Flexbox, cette mise en forme est très rapide, particulièrement efficace et responsive.

3. Les règles CSS générales

Voici les règles CSS pour l’affichage général de la page :

body {  
    background-color: #eee;  
}  
.centre {  
    width: 75%;  
    margin: 0 auto;  
}

Avec le sélecteur <body>, nous appliquons un arrière-plan à la page de couleur gris clair, avec background-color: #eee.

Le sélecteur .centre est utilisé par les éléments <header> et <main>. Il permet d’avoir une largeur d’affichage égale à 75 % de la largeur de la fenêtre du navigateur, avec width: 75%. Bien sûr, cette valeur est purement arbitraire, vous indiquez celle que vous souhaitez. L’ensemble est centré horizontalement dans la fenêtre du navigateur, avec margin: 0 auto.

Voici les règles CSS pour la mise en forme de l’en-tête :

header {  
    background-color: #000;  
    text-align: center;  
    color: #FFF;  
}  
h1 {  
    padding: 10px;  
}

L’en-tête <header> possède un arrière-plan noir, background-color: #000. Le texte est centré, text-align: center, et est de couleur blanche, color: #FFF.

Le titre <h1> possède un remplissage de 10 pixels, padding: 10px.

Voyons maintenant la mise en forme des articles :

article { 
    background-color: #FFF; 
    margin-bottom: 10px; 
} 
a { 
    text-decoration: none; 
    color: #000; 
} 
figure { 
    margin: 0; 
} 
img { 
    max-width: 100%; 
} 
figcaption { 
    text-align: center; 
    font-style: italic; 
} 
h2{ 
    margin-top: 5px; 
    padding: 5px; 
    text-align: center; 
    font-size: 1em; 
    background-color: #ccc; 
} 
p { 
    padding: 10px; 
    background-color: #FFF; 
}

Les articles, éléments <article>, possèdent un arrière-plan blanc, background-color: #FFF et une marge inférieure de 10 pixels, margin-bottom: 10px, afin d’avoir un espace avec les cartes des lignes suivantes.

Les liens <a> ne sont pas soulignés, text-decoration: none et sont de couleur noire color: #000.

L’élément <figure> permet d’afficher la photo et sa légende. Pour être parfaitement placé en haut de l’élément <article>, sans espace, les marges sont supprimées, margin: 0.

Il faut que les photos placées dans l’élément <img> puissent être redimensionnées proportionnellement par rapport à l’espace disponible. C’est pour cela que nous utilisons la propriété max-width: 100%.

C’est l’élément <figcaption> qui permet l’affichage des légendes des photos. Leur mise en forme est simple : un texte centré, text-align: center, et écrit en italique, font-style: italic.

Les titres des articles sont des éléments <h2>, avec une marge haute de 5 pixels (margin-top: 5px), un remplissage interne de 5 pixels (padding: 5px), un texte centré (text-align: center), une taille de caractère un peu plus petite (font-size: 1em) et un arrière-plan gris clair (background-color: #ccc).

Pour terminer, les textes sont placés dans des éléments <p>, avec un remplissage interne de 10 pixels (padding: 10px) et un arrière-plan blanc (background-color: #FFF).

4. La mise en page flexible et responsive

Abordons maintenant la mise en page flexible et responsive. Nous allons gérer trois largeurs d’écran : inférieure à 600 pixels pour les petits écrans, de 600 à 1200 pixels pour les largeurs moyennes comme les tablettes, et supérieure à 1200 pixels pour les écrans d’ordinateur. Bien sûr, ces valeurs sont arbitraires, à vous de les adapter à vos besoins.

Nous avons donc deux requêtes de média :

@media screen and (min-width: 600px){ /* Taille d'écran moyenne */  
    ... 
}  
@media screen and (min-width: 1200px){ /* Taille d'écran grande */  
    ... 
}

Commençons par la première requête de média. Voilà les règles CSS que nous y plaçons :

@media screen and (min-width: 600px){ /* Taille d'écran moyenne */  
    .cartes {  
        display: flex;  
        flex-wrap: wrap;  
        justify-content: space-between;  
    }  
    .carte {  
        flex: 0 0 48%;  
    }  
}

Détaillons la première règle .cartes qui s’applique à l’élément parent de tous les articles, l’élément <main class="centre cartes">.

  • display: flex : nous choisissons bien sûr un affichage flexible, avec la propriété flex-direction: row par défaut. Cela permet un affichage sur l’axe principal horizontal.

  • flex-wrap: wrap : autorise le passage à la ligne des articles, si l’espace disponible le permet.

  • justify-content: space-between : permet à l’espace supplémentaire d’être uniformément réparti entre les éléments <article> enfants.

La deuxième règle, .carte, s’applique à chacun des éléments <article class="carte"> enfants.

  • flex: 0 0 48% est la syntaxe courte des trois propriétés flex-grow, flex-shrink et flex-basis. Nous n’autorisons pas les enfants à s’agrandir, 0, ni à se réduire, 0, et la largeur initiale souhaitée est de 48%.

Cela implique que nous aurons deux colonnes et que chaque article occupera un peu moins de la moitié de la largeur disponible par rapport à l’élément parent <main>. Ainsi, nous réservons un espace de 4 % entre les colonnes : 48+48+4=100. À nouveau, libre à vous de choisir les valeurs que vous souhaitez, sans oublier que vous pouvez même utiliser si besoin est, des calculs avec la propriété calc() (https://www.w3.org/TR/css-values-4/#calc-notation).

Étant donné que nous utilisons des unités relatives en pourcentage, cela implique que les articles auront des largeurs variables et proportionnelles en fonction de la largeur de la fenêtre du navigateur.

Voici une largeur maximale :

Et voici une largeur minimale :

Pour les tailles d’écran les plus grandes, nous n’avons qu’à modifier la valeur d’affichage des articles :

@media screen and (min-width: 1200px){ /* Taille d'écran grande */  
    .carte {  
        flex: 0 0 32%;  
    }  
}

Tout d’abord, précisons qu’il n’est ni nécessaire ni utile d’indiquer à nouveau que nous voulons un affichage flexible. C’est déjà précisé dans la requête précédente.

Dans la propriété flex, nous n’autorisons toujours pas l’agrandissement et la réduction des tailles. Par contre, nous indiquons une largeur initiale souhaitée à 32%. Avec les mêmes principes que précédemment, nous avons trois colonnes et des espaces de 4 % entre chaque colonne.

Voici une largeur maximale :

Et voici une largeur minimale :

Pour les tailles d’écran les plus petites, comme sur les smartphones, la mise en page s’adapte automatiquement sur une seule colonne.

5. La gestion des liens

Il nous faut maintenant résoudre un problème d’expérience utilisateur dans l’utilisation des liens <a> présents dans la structure des articles. Pour rappel, voici la structure que nous avons :

<article class="carte">  
    <a href="#">  
        <figure>  
            ...  
        </figure>  
        <div class="contenu">  
            <h2>Nullam Sollicitudin Tortor</h2>  
                <p>Cum sociis natoque...</p>  
        </div>  
    </a>      
</article>

L’élément <a> contient tous les autres éléments constitutifs des articles. Donc le lien doit s’utiliser sur l’ensemble de ces éléments. Mais les liens ne fonctionnent que sur des contenus, que ce soit des images ou du texte. Par contre s’il n’y a pas de texte, mais une surface vide, le lien ne peut fonctionner.

Le pointeur de lien apparaît bien au survol des photos :

Et aussi au niveau du texte :

Par contre au-dessus d’une surface vide, nous n’avons pas de lien actif :

Pour résoudre ce problème, nous allons appliquer un affichage flexible à nos articles, <article class="carte">. Ainsi les éléments enfants de ce conteneur, le seul élément <a> dans cette structure, auront automatiquement un comportement flexible. Et par défaut toute la surface des éléments enfants flexibles est utilisée. Donc par conséquent, toute la surface de l’élément <a> sera active pour l’utilisation des liens.

Donc ajoutons cette propriété à la règle .carte :

.carte {  
    flex: 0 0 48%;  
    display: flex;  
}

Maintenant, toute la surface des articles est utilisable pour les liens :

Pour ajouter une petite interaction au survol des liens, nous pouvons ajouter cette règle CSS dans les règles communes :

.carte:hover {  
    box-sizing: border-box;  
    border: 2px solid #aaa;  
}

Nous ajoutons une bordure pleine de 2 pixels d’épaisseur et de couleur grise, border: 2px solid #aaa. Pour éviter le décalage engendré par cette bordure, il faut changer le mode d’affichage et de calcul des dimensions des boîtes avec box-sizing: border-box.
